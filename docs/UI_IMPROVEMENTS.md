# UI Improvements

작성일: 2026-02-22

본 문서는 오버엔지니어링 없이 기존 UI/UX를 개선한 내역을 기록합니다.

## 변경 목적
모바일 디바이스(아이폰 노치, 홈 인디케이터 등) 환경에서 웹앱을 네이티브 앱처럼 사용할 때 발생하는 화면 하단 가림 문제를 해결하기 위함입니다. `safe-area-inset-bottom`을 활용하여 시스템 안전 영역을 보장하고, 하드코딩된 패딩 값을 제거하여 컴포넌트 간 역할을 명확히 분리했습니다.

## 개선 사항 상세

### 1. 전역 레이아웃 (AppShell)
- **파일**: `src/components/layout/index.tsx`
- **변경 내용**: `AppShell` 내부의 스크롤 컨테이너 패딩 하단 값을 모바일 환경의 안전 영역과 탭바 높이를 고려하여 동적으로 계산하도록 수정.
- **수정 전**: `pb-[70px]`
- **수정 후**: `pb-[calc(70px+env(safe-area-inset-bottom))]`
- **효과**: 아이폰 등 하단 인디케이터가 있는 기기에서도 리스트 하단 콘텐츠가 탭바에 가려지지 않고 온전히 스크롤 됨.

### 2. 페이지 레벨 하단 패딩 중복 제거
- **대상 파일**:
  - `src/app/(app)/feed/page.tsx`
  - `src/app/(app)/mypage/page.tsx`
- **변경 내용**: `min-h-full pb-20`로 고정 적용되어 있던 패딩을 제거.
- **효과**: `AppShell`이 하단 여백 관리를 책임지게 되어, 개별 페이지 컴포넌트에서 중복으로 여백을 관리할 필요가 없어짐. (유지보수성 향상)

### 3. 지도(Map) 뷰 하단 패딩 안전 영역 적용
- **파일**: `src/app/(app)/map/page.tsx`
- **변경 내용**: 리스트 뷰 영역의 하단 패딩에 모바일 안전 영역 동적 변수 추가.
- **수정 전**: `pb-24 lg:pb-4`
- **수정 후**: `pb-[calc(24px+env(safe-area-inset-bottom))] lg:pb-4`
- **효과**: Map 화면의 리스트뷰 최하단 마커 카드가 안전 영역 아래로 숨는 문제 방지.

## 결론
별도의 외부 라이브러리나 복잡한 상태 관리 추가(오버엔지니어링) 없이, CSS `env()` 변수와 공통 레이아웃 아키텍처를 재조정하여 모바일 웹앱 UX 품질을 네이티브 수준으로 끌어올렸습니다.
